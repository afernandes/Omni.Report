Por que OmniReport

Diferenciais que importam

⚖️

Open-source, MIT

Sem royalties, sem licença por desenvolvedor/servidor/deploy. Código auditável e forkável — diferente de DevExpress, Telerik e Stimulsoft.

🔁

3 modos, 1 modelo

Code-first fluente, records imutáveis e Designer Blazor produzem o mesmo ReportDefinition e a mesma saída. Não é "designer com export-only".

🟣

100% .NET, cross-platform

SkiaSharp para render vetorial; roda em Blazor Server/WASM e MAUI Hybrid (Windows + Android). Sem amarras de plataforma.

🧾

Nichos raros de saída

ESC/POS térmico (NFC-e/DANFE, 58/80 mm), Markdown GFM e JSON determinístico para snapshot/RAG — além de PDF/XLSX/DOCX/HTML.

↪️

Migração de SSRS

Importa .rdl real (~80%) convertendo expressões VB-SSRS, com avisos explícitos de perda — nunca descarte silencioso.

🧩

Extensível sem atrito

Serialização auto-wired por convenção: um elemento novo faz round-trip sem editar serializers. Interfaces limpas para dados, exporters e impressoras.

Autoria

Os três modos

O mesmo relatório, escrito do jeito que faz sentido para o seu time.

var report = ReportBuilder.Create("Vendas")
    .Page(p => p.A4().Portrait().Margins(20))
    .DataSource("Vendas", vendas)
    .ReportHeader(h => h.Height(15)
        .Text("Relatório de Vendas").At(0,0).Size(170,12).Bold().Center())
    .Group("PorCliente", "Fields.Cliente", g => g
        .Header(h => h.Text("Cliente: {Fields.Cliente}").At(0,1).Bold())
        .Detail(d => d
            .Text("{Fields.Produto}").At(0,0).Size(100,6)
            .Text("{Fields.Total:C}").At(140,0).Size(30,6).AlignRight()))
    .ReportFooter(f => f.Text("Total: {Sum(Fields.Total):C}").Bold().AlignRight())
    .Build();

var rendered = await report.PaginateAsync();
new SkiaPdfExporter().ExportToFile(rendered, "vendas.pdf");
// Modelo imutável — ideal para geração programática / layouts computados
var def = new ReportDefinition("Etiqueta", PageSetup.A4Portrait,
    detail: new DetailBand(Unit.FromMm(20), new EquatableArray<ReportElement>(
        new LabelElement { Text = "Produto",

        new BarcodeElement { Symbology = BarcodeSymbology.QrCode,
            Expression = "Fields.Sku",

Recursos

Tudo que você precisa para um relatório

Elementos

Label, TextBox (multi-estilo), Line, Rectangle (container), Ellipse, Image, Chart (8 tipos), Tablix (tabela + matrix/pivô), Barcode/QR, Subreport, Code.

KPIs & viz

Gauge radial/linear, DataBar, Sparkline, Indicator, mapa vetorial (Web Mercator + graticule + GeoJSON + tiles OSM).

Bandas & grupos

Report/Page/Group/Detail/SubDetail headers e footers, master-detail, KeepTogether, repeat-header, CanGrow/CanShrink.

Estilo & binding

Fonte/cor/borda/padding/format, cores nomeadas CSS, PropertyExpressions (qualquer prop), formatação condicional, ações (link/bookmark/drillthrough).

Dados

IEnumerable, DataTable, SQL (SQLite/PostgreSQL/SQL Server/MySQL/ Oracle via AdoNet), JSON/XML/REST/FileSystem, campos calculados, relações, parâmetros tipados.

Expressões

NCalc + VB-SSRS: agregados com escopo (Sum/Avg/Var/StDev…), Lookup/LookupSet/MultiLookup, posicionais, IIf/Switch, funções de texto/data/formatação.

Paginação

Duas passadas, multi-coluna (snake), PrintOnFirst/LastPage, CanShrink que encolhe a banda, split de banda por elemento.

Impressão

Spooler Windows (duplex/bandejas/print-to-PDF) e ESC/POS térmico (Bematech/Daruma/Elgin/Epson) — USB/serial/rede.

Arquitetura

Modelo imutável → primitivas → qualquer backend

autoriaReportDefinition → PaginateAsync()RenderedReport → primitivas planasLayoutPrimitive[] → exporter / rendererPDF · DOCX · PNG · …

Núcleo puro

Reporting.Core não depende de plataforma nem de UI. Bandas e elementos são sealed record imutáveis com igualdade por valor.

Plug-and-play

Renderers (Skia/GDI), exporters e impressoras são intercambiáveis via DI (IRenderingContext / IReportExporter / IReportPrinter).

UI reutilizável

Designer e Viewer são Razor Class Libraries — o mesmo binário roda em Blazor Server, Web App e MAUI Hybrid.
